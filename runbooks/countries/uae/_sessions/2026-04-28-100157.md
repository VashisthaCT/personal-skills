## Retraction
- **Wrong claim:** "UAE was on `GCC_EINVOICE` in MEA region; got re-bound to `EINVOICE_GLOBAL` during EU migration."
- **Contradicting evidence:** `pdfgenerator` PR #414 (commit `a5f8afc7 Changes for UAE einvoice schema`, Nov 4 2025, Ashraf Nawaz) explicitly added UAE-specific fields (`sellerSupplierParty`, `taxRepresentativeParty`, `discrepancyResponse`, `profileExecutionID`) to `EInvoiceGlobalDocumentData.kt`. UAE has been on `EINVOICE_GLOBAL` since at least Nov 2025 — well before any AWS region migration.
- **Corrected diagnosis:** the bug is env-var config drift on `WHOLE_NUMBER_PERCENT_COUNTRY_CODES`. einvoicing-core has a country-aware percent-format normalizer (`PrintSchemaMapper.convertPercentForPrint`) that divides 5 → 0.05 when the country is in this set. AE was missing from the Vault override in EU region → percent stayed at 5 → pdfgenerator multiplied × 100 → 500 rendered.
- **Lesson:** binding/payload-type/routing claims need git log evidence — never inferred from convention. Codified in `SKILL.md` "Anti-patterns observed in past sessions" #1.

## Answer (final summary)
einvoicing-core has a country-aware percent-format normalizer: `PrintSchemaMapper.convertPercentForPrint(percent, isPercentInWholeNumberFormat)` divides by 100 when flag=true. Flag set from `print-service.wholeNumberPercentCountryCodes` (Vault env-driven). UAE expects this list to include `AE`. YAML defaults are inconsistent across modules (`application/`: `AE,BE`; `einvoicing-workflow-consumer/` and `einvoicing-temporal-worker/`: `BE` only). The Vault override for the EU region was missing `AE`. Fix: add `AE` to Vault key `WHOLE_NUMBER_PERCENT_COUNTRY_CODES`. **Confirmed working by user Apr 28 2026.**
